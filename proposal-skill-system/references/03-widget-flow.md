# ウィジェットフロー（方針確認の標準）

各STEPの終了時に `ask_user_input_v0` ツールでユーザーの判断を仰ぐ。判断後、次のSTEPに進むか、修正するか、再生成するかを選択する。

---

## widget_after_outline （STEP 1: 骨子生成後）

### 提示順序

1. 骨子のMarkdownテキストを **会話に表示**（19枚分、各スライドの「論点・タイトル・メインメッセージ・チャートイメージ」）
2. その下に短い要約を1文添える（例: 「19枚構成、最重要施策は施策3、ROIは標準シナリオで+246万円/年」）
3. その上で `ask_user_input_v0` を呼び出す

### ウィジェット質問

```python
ask_user_input_v0(
    questions=[
        {
            "question": "骨子の方針は問題ないですか？",
            "options": [
                "進める（HTMLDeck生成へ）",
                "一部修正したい",
                "大幅に作り直したい"
            ],
            "type": "single_select"
        }
    ]
)
```

### 分岐後の処理

| 選択 | 処理 |
|---|---|
| 進める | STEP 2に進む |
| 一部修正したい | 自由記述で「どのスライドのどこを修正するか」を聞き直す（**追加の ask_user_input_v0 は呼ばず**、テキストで質問） |
| 大幅に作り直したい | 「どの方針を変えたいですか？（章立て／メッセージ／チャート種別／その他）」をテキストで聞く |

---

## widget_after_html （STEP 2: HTMLDeck生成後）

### 提示順序

1. HTMLDeckを `present_files` で表示
2. 主要なデザインポイント・コンテンツチェック結果を箇条書きで添える（例: 「全Pメッセージ120字以内 / 5seg値は◎○△で表現 / CS連番付与 / KPIツリーP6=P15同一」）
3. その上で `ask_user_input_v0` を呼び出す

### ウィジェット質問

```python
ask_user_input_v0(
    questions=[
        {
            "question": "HTMLDeckのデザイン・内容は問題ないですか？",
            "options": [
                "進める（PPT生成へ）",
                "スライド内容を一部修正",
                "デザインを修正",
                "deck.mdを読み込んで再生成"
            ],
            "type": "single_select"
        }
    ]
)
```

### 分岐後の処理

| 選択 | 処理 |
|---|---|
| 進める | STEP 3に進む |
| スライド内容を一部修正 | 「どのスライドのどこを？」をテキストで聞き、ピンポイント編集（consulting-slide-builderの編集モードと同じ） |
| デザインを修正 | 「色味／タイポ／レイアウト／余白／その他」を聞いてCSS変数を調整 |
| deck.mdを読み込んで再生成 | ユーザーがdeck.mdを送ってくるのを待ち、design-systemを上書きして v2 を生成 |

---

## STEP 3 終了時（ウィジェット不要）

PPT生成完了後はウィジェットを呼ばない。代わりに：

1. `present_files` で PPTX と HTMLDeck を **並べて** 提示
2. 一言メッセージを添える：

> 完成です！HTMLDeckとPPTXを並べて出力しました。PPTXは編集可能なテキストボックス・図形で構成されているので、PowerPoint で開いて部分修正できます。何か変更があれば声をかけてください。

---

## ウィジェット運用の原則

### 質問は1つに絞る

3つまで聞けるが、3つ並べると認知負荷が上がるので **基本は1問**。「進める/修正/作り直す」で十分。

### 選択肢は「次のアクション」を示す

- ✅ 「進める（HTMLDeck生成へ）」← 何が起きるか明確
- ❌ 「OK」「問題なし」← 次のアクションが不明

### ウィジェット直前に必ず本体を提示

ウィジェットだけ単発で出さず、必ず **直前に骨子テキスト or HTMLファイル** を見せた上で質問する。「これを見てどう思いますか？」のフロー。

### ウィジェット内では結論を急がない

「ジャストアイディアですが」「目的を取り違えていたらご指摘ください」のクッション言葉を、ウィジェット直前のメッセージに添える。
