# 19ページ構成テンプレート（デフォルト）

提案書の標準構成。案件によって省略・追加・順序変更が可能。

## 全体ストーリー

```
前提（BM/3C） → 課題（経営/KPI） → 原因（顧客体験/エラー） → 打ち手（4施策） → 推進（ROI/ロードマップ）
```

5ブロック構成で、論点の順序に沿って意思決定材料を一気通貫で揃える。

## 案件タイプ別の調整パターン

### F-1型（D2C・サブスク）
- P5の「F0→F1の壁」→「F0→F1の壁」（同じだが意味は試用→定期化）
- P6の主戦場 → 「定期化率」または「2回目購入率」
- P7のCSステージ → 認知/興味/初回購入/2回目/定期化/離反

### F-3型（B2B SaaS）
- P5の「F0→F1」→「トライアル→有料化」
- P6の主戦場 → 「トライアル→有料CVR」または「アクティベーション率」
- P10〜P13 施策 → オンボーディング/ヘルススコア/CS介入/エクスパンション

### サービス業（美容/飲食/医療）
- P5の「F0→F1」→「初回来店→2回目来店」
- P6の主戦場 → 「2回目来店率」または「リピート率」
- P10〜P13 施策 → 来店時UX/フォロー連絡/特典設計/オフライン体験
