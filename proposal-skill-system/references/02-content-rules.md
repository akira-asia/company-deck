# コンテンツルール（厳守）

提案書の品質を担保する4つの絶対ルール。違反した場合は生成し直す。

---

## ルール1: メインメッセージは120字以内

各スライドの `<div class="slide-subtitle">` の中身は、強調`<span>`を含めて **120字以内** に収める。

### NG例（200字超）

```
プラットフォーム型一般論の勝ち筋3要件と典型ボトルネックを整理した上で、貴社の5seg推測値（F0:75〜80%、F1:5〜8%、F2:6〜8%、ロイヤル:3〜5%）に降ろし、F0→F1転換率15〜20%が業界合格20%未達の危険水準にある仮説を可視化する。各5segの定義文と推測値の幅を併記することで、ヒアリング前段階の前提を透明化する。
```

### OK例（103字）

```
PF型の勝ち筋3要件と典型ボトルネックに5seg推測値を降ろし、F0→F1転換率が業界合格水準未達（△水準）の危険水域にある仮説を可視化する。 → F0→F1の壁が最大課題、ロイヤル比率の低さが次の構造課題
```

### 圧縮テクニック

- 「[正式名称]」→「[略称]」（プラットフォーム→PF、カスタマージャーニー→CS など）
- 「〜することで」「〜となる」→ 削除して名詞止め
- 重複説明（「3軸（5seg・CS・顧客体験）」のような括弧内補足）→ 削除
- 「考えられる」「と整理できる」「という構造を俯瞰する」→ 「〜する」「〜と位置づける」に圧縮

---

## ルール2: 強調`<span>`は本当に効かせたい1点だけ

メインメッセージ内の強調（`<span style="color:var(--accent); font-weight:700;">→ ...</span>`）は、**各スライド最大1箇所**。装飾的な強調は使わない。

### 強調すべきケース
- 結論の一言（「→ 主戦場はKSF-2（CVR）」）
- アクション指示（「→ 施策の力点はここに置く」）
- 注目すべき定量効果（「→ 4施策合計で年間+112万〜+1,315万円」）

### 強調すべきでないケース
- メインメッセージで既に述べた内容のリピート
- 当たり前の説明
- 装飾としての強調

### 強調を削除すべきページ例
- アジェンダ → 強調なし
- エグゼクティブサマリ → 強調なし（メッセージ自体が結論）

---

## ルール3: 5seg推測値は◎/○/△で含み表現

5segの構成比・転換率・売上貢献率などの **推測値** は、具体％を直接引用せず、以下の記号で水準感のみ表現する。

### マッピングルール

| 元の数値 | 含み表現 | 意味 |
|---|---|---|
| 構成比 70%以上 | ◎（最大層） | 圧倒的多数 |
| 構成比 30-69% | ○ | 中位 |
| 構成比 5-29% | △ | 少数 |
| 構成比 5%未満 | △（危険水域） | ほぼゼロ |
| 転換率 業界理想以上 | ◎ | 理想水準 |
| 転換率 業界合格水準内 | ○ | 合格 |
| 転換率 業界合格水準未達 | △（危険水準） | 改善必須 |
| 単価 同型平均超 | ◎ | 高水準 |
| 単価 同型平均水準 | ○水準 | 標準 |
| 単価 同型平均未満 | △ | 低単価 |

### 業界基準値はそのまま残す

「合格20-35%」「理想45%+」「合格10-20%／5%は危険」のような **業界一般のベンチマーク** は具体％のまま残す（これは引用ではなく定義）。

### 適用例

#### 元のテキスト
```
F0:75〜80%、F1:5〜8%、F2:6〜8%、ロイヤル:3〜5%、F0→F1転換率15〜20%が業界合格20%未達
```

#### 含み表現後
```
F0:◎（最大層）、F1〜ロイヤル:△、F0→F1転換率は△水準（業界合格水準20-35%帯未達）
```

### 適用範囲

- スライドのメインメッセージ
- SVG内の数値テキスト（`<text>` 要素内）
- 結論バー・注記の本文

## ルール4: CS連番と顧客体験カテゴリ連番

CSと顧客体験カテゴリは、表・カード・ラベルすべてに連番を必ず付与する。

### CS連番（顧客ステージ7段階）

| 番号 | CS名 | 該当5seg |
|---|---|---|
| 1 | 認知 | 未顧客 |
| 2 | 会員登録済み未利用 | 未顧客（F0） |
| 3 | 比較・検討 | 未顧客 |
| 4 | 初回予約完了（申込・決済） | トライアル（F1） |
| 5 | 継続利用 | 一般顧客（F2） |
| 6 | ロイヤル | ロイヤル顧客 |
| 7 | 離反 | 離脱顧客 |

### 顧客体験カテゴリ連番（10ステージ）

| 番号 | カテゴリ名 |
|---|---|
| 1 | サイト訪問 |
| 2 | 会員登録 |
| 3 | 条件絞り込み検索 |
| 4 | 商品詳細閲覧 |
| 5 | 申し込み・購入 |
| 6 | 実物確認・宿泊体験（オフライン） |
| 7 | アプリ起動・利用 |
| 8 | レビュー投稿 |
| 9 | 関連サービス予約 |
| 10 | メール開封・再訪 |

### 表記スタイル

- 短縮: `2. 会員登録済`、`1. 認知`、`3. 比較・検討`
- フル: `CS-2: 会員登録済未利用`、`CS-1: 認知`、`CS-3: 比較・検討`
- レンジ: `CS-2〜CS-3「会員登録済未利用〜比較・検討」`

スライドのスペースに応じて使い分け。表のセルが狭い場合は短縮形、ヘッダーやナビ的な場所はフル表記。

### 業種別の調整

- **F-1型 D2C/サブスク**: CS-3を「定期登録検討」、CS-4を「初回購入」、CS-5を「定期化」とリネーム
- **F-3型 B2B SaaS**: CS-1〜CS-7を「リード/トライアル/PoC/有料化/アクティベーション/拡張/解約」に置換
- **サービス業（美容/医療等）**: CS-2〜CS-3を「予約サイト来訪/予約検討」、CS-4を「初回来店」に置換

## ルール5: KPIツリー一貫性

複数のスライドにKPIツリーが登場する場合、それらは **完全に同一構造（KGI→KSF×4→KPI×8の3階層）** とする。

### 該当ページ

- P6（KPIツリー単独ページ）：横長レイアウト
- P15（算出式ロジック）の左カラム：縮小縦型レイアウト

### 同期すべき要素

| 要素 | 同期 |
|---|---|
| KGI名 | 同じ（例：流通取引総額） |
| KSF×4のラベル | 同じ（KSF-1〜KSF-4） |
| 主戦場のKSFと配下KPIのアクセント色 | 同じ（青 #2563EB） |
| KPI×8の名称 | 同じ |
| 基本分解式 | 同じ |
| 注記の主戦場メッセージ | 同じ |

### レイアウト調整は許容

P15は左カラム（幅370px）に圧縮するため、フォントサイズや余白は調整して構わないが、**論理構造は一切変えない**。
