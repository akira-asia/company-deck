# デザインシステム（フォールバック）

このファイルは `deck.md` が提供されていない場合のデザイン基準として使う。`deck.md` が提供された場合は **そちらを優先**し、このファイルは無視する。

---

## CSS変数（基本）

```css
:root {
  /* Colors — Base palette */
  --bg: #FFFFFF;          /* 背景白 */
  --card: #F0F0F0;        /* カード背景グレー */
  --card-mid: #E0E0E0;    /* カード中間グレー */
  --line: #999999;         /* 罫線・補助色 */
  --dark: #1A1A1A;         /* 黒テキスト・濃い背景 */
  --sub: #555555;          /* サブテキスト */

  /* Colors — Accents */
  --accent: #02BDBD;       /* アクセント（メインメッセージの強調・主戦場）/ または #2563EB（ブッキングブルー系） */
  --warn: #E04E49;         /* 警告・注目色（最大流出ポイント・危険水域）*/
  --ok: #1A1A1A;           /* OK表示（黒で控えめに）*/

  /* Spacing & Radius */
  --gap: 16px;
  --radius: 4px;
}
```

### アクセント色のルール

- 第1選択: `#02BDBD`（ターコイズ）— Booking.com案件以外で推奨
- 第2選択: `#2563EB`（ブッキングブルー）— Booking.com案件、または濃いブランドカラーが必要な案件

deck.md にカラー指定がある場合はそれを使う。

## レイアウトテンプレート（10種）

`consulting-slide-builder` の `references/html-template-system.md` と互換。主要なものを再掲：

### Layout 01: Cover
- 中央寄せタイトル
- サブタイトル＋日付＋自社名
- フッター不要

### Layout 02: Agenda
- 縦並び章リスト
- 各章ボックス（章番号 + タイトル + 1行説明）

### Layout 03: Executive Summary
- 課題（仮説）/ 原因（仮説）/ 打ち手 の3行バー
- 各バー左端は黒帯ラベル、右側は白カード
- 打ち手のラベル背景はアクセント色

### Layout 04: KPI
- 4分割または5分割グリッド
- 各セルに数値（大）+ ラベル + サブテキスト

### Layout 05: 2カラム
- 左右等分
- 左: ビジュアル（アイコン/図解）、右: テキスト
- 比率は 40:60 または 50:50

### Layout 06: 3カラム
- 横3等分
- 各カラムに小タイトル + 本文 + アクセント

### Layout 07: Process Flow
- 横向き矢印で連結された4-5ステップ
- 各ステップは番号 + タイトル + 短文

### Layout 08: Chart
- スライド領域いっぱいにSVG概念図
- viewBox は `0 0 800 380` または `0 0 1000 500`

### Layout 09: Next Steps
- ToDo形式の3-5項目
- 各項目: 担当 + 期限 + 内容

### Layout 10: Section Divider
- 章区切り
- 中央に大きな章番号 + 章タイトル

## SVG概念図のルール

### viewBox

| 用途 | viewBox |
|---|---|
| 標準スライド（slide-subtitleが上部にある） | `0 0 800 380` |
| フル画面で見せたい大型図解 | `0 0 1000 500` |

### 配色ルール

- ベース: `#F0F0F0` / `#E0E0E0` / `#999999` のグレー基調
- アクセント: `var(--accent)`（主戦場・最重要）
- 警告: `var(--warn)`（最大流出ポイント・危険水域）
- 文字色: `#1A1A1A`（基本）/ `#555555`（サブ）/ `#888888`（注記）

### 装飾ルール

- シャドウ・グラデーションは **使わない**
- 枠線は最小限（`stroke-width="1"` または `0.5`）
- アクセント要素のみ `stroke-width="2"` を許可

### テキスト折り返し

- 文字数が多い場合は `<tspan x="..." dy="14">` で2行に
- フォントを下げて1行に収めようとしない（最小11px厳守）

## アクセント色を使う場所（ガイド）

| 場所 | アクセント | 警告 |
|---|---|---|
| メインメッセージ強調`<span>` | ✅ | — |
| 主戦場のKSF枠 | ✅ | — |
| 主戦場配下のKPI枠 | ✅ | — |
| 最大流出ポイントの課題コード枠 | — | ✅ |
| 危険水域の数値（赤字） | — | ✅ |
| 矢印（転換ボトルネック） | — | ✅ |
| 結論バー（最下部） | ✅ | — |
| ★最重要施策の枠 | ✅ | — |

**過剰使用禁止**：1スライドにつき、アクセントは2-3箇所、警告は1-2箇所が上限。

## フッター（全スライド共通）

```html
<div class="slide-footer">
  <span>2026. ○○ Company</span>
  <span>N / [総ページ数]</span>
</div>
```

- 左: 自社名・年度
- 右: ページ番号 / 総ページ数

## deck.md が来たときの上書き手順

ユーザーから `deck.md` が提供されたら：

1. `deck.md` を `view` で読み込む
2. 以下を上書き対象とする：
   - CSS変数の値（カラー）
   - タイポグラフィ（font-family, サイズ）
   - 各レイアウトの細部（余白・角丸・罫線）
3. **論理構造（19枚構成・コンテンツルール・連番ルール・含み表現）は上書きしない**

deck.md は「見た目」を定義するもので、「論理」は定義しない。両者を混同しない。
